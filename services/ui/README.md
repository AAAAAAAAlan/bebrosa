# HappinessMP UI

TanStack Start UI with Discord authentication powered by the Better Auth API.

## Commands

Run these commands from the repository root:

```bash
bun run ui:dev
bun run ui:build
bun run ui:test
bun run ui:lint
bun run ui:check
```

## Structure

- `src/routes/__root.tsx` contains the HTML shell and global metadata.
- `src/routes/index.tsx` is the empty application entry route.
- `src/styles.css` contains only Tailwind and a small browser reset.
- `src/routeTree.gen.ts` is generated by TanStack Router; do not edit it manually.

Add application routes under `src/routes` and shared UI under `src/components` as needed.

The development UI runs on <http://localhost:3001> and proxies `/api` requests
to the API on <http://localhost:3000>. Set `VITE_AUTH_BASE_URL` when the auth API
is hosted on another origin. That origin must also be listed in the API's
`BETTER_AUTH_TRUSTED_ORIGINS` setting, and the deployment must allow
credentialed CORS requests from the UI origin.

For an in-game WebUI, build the app first and expose the generated client files through the HappinessMP resource `meta.xml` before loading its HTML entry point with `WebUI.Create`.
